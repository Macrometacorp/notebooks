Py06 - Query Workers#

Query as API (aka Query Workers) enables developers to convert saved C8QL queries into geo-distributed REST APIs quickly. Query Workers eliminate the need for separate backend servers and containers for CRUD operations.

Pre-requisite#

Let’s assume your

  • Tenant name is an email address

  • User password is xxxxx

If you need to install pyC8, you can run the cell below; otherwise, you may skip it.

!pip install pyC8

1. Import Libraries#

import json
import getpass

2. Import c8Client#

from c8 import C8Client

3. Define Login Credentials#

geo_fabric = "_system"

fed_url = input("Please type the federation host, leave it blank to use the default and press enter:") or "gdn.paas.macrometa.io"

guest_mail = input("Please, type your email and press enter:")

guest_password = getpass.getpass("Please, type your password and press enter:")

4. Connect to GDN#

Now that you have added your login credentials, let’s connect to GDN:

print("\n ------- CONNECTION SETUP  ------")
print("tenant: {}, geofabric:{}".format(guest_mail, geo_fabric))

try:
    client = C8Client(protocol='https', host=fed_url, port=443,
                    email=guest_mail, password=guest_password,
                    geofabric=geo_fabric)
    print("Login successfully")
except Exception as err:
    print("Error while login: ", err)

5. Create a Geo Replicated Collection#

Before we create the Query Workers, let’s create a collection that we can work with:

collection_name = "person"

print("\n ------- CREATE GEO-REPLICATED COLLECTION  ------")
if client.has_collection(collection_name):
    print("Collection exists")
else:
    employees = client.create_collection(collection_name)
    
print("Created collection: {}".format(collection_name))

6. Create Query Workers#

6.1. Defining our Queries#

In this section, we are going to define five Query Worker APIs:

value = "INSERT {'firstname':@firstname, 'lastname':@lastname, 'email':@email, 'zipcode':@zipcode, '_key': 'key101'} IN %s" % collection_name
parameter = {"firstname": "", "lastname": "", "email": "", "zipcode": ""}

insert_data = {"query": {"name": "insertRecord", "parameter": parameter, "value": value}} 
get_data = {"query": {"name": "getRecords", "value": "FOR doc IN %s RETURN doc" % collection_name}}
update_data = {"query": {"name": "updateRecord", "value": "UPDATE 'key101' WITH { \"lastname\": \"cena\" } IN %s" % collection_name }}
delete_data= {"query": {"name": "deleteRecord", "value": "REMOVE 'key101' IN %s" % collection_name}}
get_count = {"query": {"name": "countRecords", "value": "RETURN COUNT(FOR doc IN %s RETURN 1)" % collection_name}}

6.2. Creating the Query Worker APIs#

The Following Code will create a set of Query Workers in GDN. Once complete, you can check that they exist from your console access, under Queries, and navigate to “Saved Queries”.

print("\n ------- CREATE RESTQLs  ------")
client.create_restql(insert_data)  # name: insertRecord
client.create_restql(get_data)  # name: getRecords
client.create_restql(update_data)  # name: updateRecord
client.create_restql(delete_data)  # name: deleteRecord
client.create_restql(get_count)  # name: countRecords
  
print(json.dumps(client.get_restqls(), indent=4))

7. Execute Query Workers#

Note: We will skip testing the deleteRecord API in this section to demonstrate how to update a Query Worker API first.

7.1. Test the “insertRecord” Query Worker#

print("\n ------- EXECUTE RESTQLs ------")
print("Insert data....")
response = client.execute_restql(
    "insertRecord",
    {"bindVars": {"firstname": "john", "lastname": "doe",
                    "email": "john.doe@macrometa.io", "zipcode": "511037"}})

print("Get data....")
response = client.execute_restql("getRecords")

print(json.dumps(response, indent=4))

7.2. Test the “updateRecord” Query Worker#

print("Update data....")
response = client.execute_restql("updateRecord")

print(json.dumps(response, indent=4))

7.3. Test the “getRecords” Query Worker#

print("Get data....")
response = client.execute_restql("getRecords")

print(json.dumps(response, indent=4))

7.4. Test the “countRecords” Query Worker#

print("Count records....")
response = client.execute_restql("countRecords")

print(json.dumps(response, indent=4))

8. Updating a Query Worker API#

8.1. Test that we can query GDN and see the document#

A common requirement when building an application is to make a change to an API and update it.

To demonstrate this, let’s first query the database using the execute_query method we learned in an earlier section.

Once run, we should see the document in the cell output in JSON format so we can read it easily.

print("Get docs in \"person\" collection")

cursor = client.execute_query('FOR doc IN person RETURN doc')

docs = [document for document in cursor]

print(json.dumps(docs, indent=4))

8.2. Updating a Query Worker API#

The following cell shows us how to add a new parameter to the Query; in this example, we will add the parameter “phone” to the query.

value = f"INSERT {{'firstname':@firstname, 'lastname':@lastname, 'email':@email, 'zipcode':@zipcode, 'phone': @phone}} IN {collection_name}"
parameter = {"firstname": "", "lastname": "", "email": "", "zipcode": "", "phone": ""}

insert_data = { "query": { "parameter": parameter, "value": value } } 

query_name = "insertRecord"
response = client.update_restql(query_name, insert_data)
print(response)

8.3. Testing the updated Query Worker API#

Now that we have updated the query, let’s add a new record to test that it works:

print("Insert data with updated query worker")
response = client.execute_restql(
    query_name,
    {"bindVars": {"firstname": "john", "lastname": "doe",
                    "email": "john.doe@macrometa.io", "zipcode": "511037", "phone": "213-555-9578"}})
print(response)

8.4. Checking the data in the collection#

Now, let’s check that the data has been updated in the record by using the execute_query method - we should see two documents.

  1. The original document without a phone number, and

  2. A second document, but now with a phone number added

Hint: If you would like to return to the step “countRecords” and re-run the code, the count value should increase by 1.

print("Get docs in \"person\" collection, after executing the updated query")

cursor = client.execute_query('FOR doc IN person RETURN doc')

docs = [document for document in cursor]

print(json.dumps(docs, indent=4))

9. Deleting a record using a Query Worker API#

print("Delete data....")
response = client.execute_restql("deleteRecord")

print(json.dumps(response, indent=4))

10. Delete all Query Workers#

We might want to delete our RESTQL APIs completely.

For each Query Worker, we want to remove, we use the delete_restql method as shown in the cell below

print("\n ------- DELETE RESTQLs ------")
client.delete_restql("insertRecord")
client.delete_restql("getRecords")
client.delete_restql("updateRecord")
client.delete_restql("countRecords")
client.delete_restql("deleteRecord")

print("\n ------- DONE  ------")

11. Delete the Collection#

Now let us remove the collection we added during this tutorial:

print("Collection Deleted: ",client.delete_collection(collection_name))
